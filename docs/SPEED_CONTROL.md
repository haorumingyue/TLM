# 速度控制策略文档

## 1. 概述

本系统采用**三条皮带级联仿真**（主运 → 斜井 → 101），结合 **PID 前馈-反馈控制**与**离散档位换挡**策略，根据入流动态调整各皮带带速，实现节能降耗与防堵保护。

```
3306综采(A) ──→ ┌──────────────┐
                │  主运大巷皮带  │ ──→ ┌──────────────┐
3217综采(B) ──→ │  5000m,水平   │     │  主斜井皮带   │ ──→ ┌────────┐
                └──────────────┘     │  1160m,提升340m│     │ 101皮带│
                                     └──────────────┘     │  147m  │
                                                          └────────┘
                                                              ↓
                                                          最终排出
```

三条皮带**直接串联**，上游出流直接注入下游皮带首格（受下游最大线密度限制），无中间缓冲队列。A/B 队列由外部日志驱动，作为主运皮带的入料缓冲。

**设计哲学**：
1. 离散档位控制：各皮带统一 5 档速度，通过死区滞回防止频繁换挡。
2. 前馈-反馈复合控制：前馈基于入流推算目标速度，反馈基于填料比修正。
3. 非对称响应：升档即时（安全优先），降档延迟（避免反复升降）。
4. 物理能耗模型：基于阻力、提升高度、辅助功率的功率计算，积分得到累计电量 (kWh)。

## 3. 调速策略架构

实现：[simulator.py](../src/core/simulator.py)

### 3.1 主运皮带 — PID + 前馈预测

接收两路工作面（A/B）入流，使用 `PIDStrategy` 计算连续目标速度，经 `_apply_gears` 映射到离散档位。同时接收 Chronos-2 / TimesFM 预测的未来流量作为前馈。

### 3.2 斜井皮带 — PID（无前馈）

入流来自主运出流（直接注入斜井首格），使用独立 `PIDStrategy` 实例。因主运出流有传输延迟，A/B 预测值不直接用于斜井前馈。

### 3.3 101 皮带 — PID + 档位

三条皮带在智能模式下均使用 `PIDStrategy` + `_apply_gears`，带速范围与档位表一致。

## 4. PID 控制算法

实现：[pid.py](../src/core/pid.py) `PIDStrategy`

### 4.1 关键参数

| 参数 | 值 | 含义 |
|------|-----|------|
| `V_MIN` | 1.5 m/s | 最低带速 |
| `V_MAX` | 4.5 m/s | 最高带速 |
| `L_OPT` | 0.60 | 目标填料比（0~1） |

### 4.2 局部拥堵检测

```python
w = np.linspace(0.2, 1.0, len(belt_load))  # 装料端权重最高
s_max = np.max(belt_load * w)
```

`belt_load` 是各格填料比（0~1），权重从出料端 0.2 线性递增到装料端 1.0。`s_max` 与 `L_OPT` 直接比较。

### 4.3 前馈控制

```python
# 预测轨迹加权衰减
decay = [1.0 / (1.0 + 0.3 * k) for k in range(len(pred_inflow))]
ref_pred = max(pred_inflow * decay)
ref = max(inflow, ref_pred)

# 推算维持目标填料比所需带速
v_flow = ref / (max_density * L_OPT)
```

- 取当前入流与加权预测的最大值作为参考流量
- 预测步按 `1/(1+0.3k)` 衰减，近端权重高，远端递减
- `v_flow = 入流 / (线密度上限 × 目标填料比)`

### 4.4 超填响应

当皮带填料比超过 `L_OPT` 时，按线性比例从 `v_flow` 向 `V_MAX` 推高：

```python
if s_max > L_OPT and v_flow < V_MAX:
    excess = (s_max - L_OPT) / L_OPT
    t = min(1.0, excess)
    v_overfill = v_flow + t * (V_MAX - v_flow)
```

### 4.5 积分与平滑

- **积分项**：消除稳态误差，目标到达时（误差 < 2%）按 `2%/s × dt` 衰减（Anti-windup）
- **低通滤波**：`alpha = dt / (2.0 + dt)`，一阶惯性平滑
- **变化率限制**：±0.15 m/s per step，防止机械冲击

## 5. 换挡逻辑

实现：[simulator.py](../src/core/simulator.py) `_apply_gears`

### 5.1 死区滞回

```
    2.25      2.5875   3.0     2.6625      3.75
     |----------|--------|---------|-----------|
            降档阈值   当前档    升档阈值
           (中点-0.05)          (中点+0.05)
```

- **升档阈值** = 相邻档中点 + **0.05 m/s**
- **降档阈值** = 相邻档中点 − **0.05 m/s**

### 5.2 非对称驻留

| 操作 | 驻留要求 | 原因 |
|------|---------|------|
| 升档 | 无（即时） | 流量突增时快速升速防堵 |
| 降档 | 30 s | 等待确认流量确实下降 |

升档后驻留计时器重置，降档需重新等待 30 s。

## 6. 物理能耗模型

### 6.1 功率计算

```
P_total = P_auxiliary + P_motion + P_lift
```

| 分量 | 公式 | 说明 |
|------|------|------|
| 辅助功率 | `aux_kw` (常量) | 驱动站空载运行 |
| 运动功率 | `C_res × (m_empty + m_coal) × g × v / η / 1000` | 克服摩擦和滚动阻力 |
| 提升功率 | `Q_kgps × g × H / η / 1000` | 物料提升功率（仅斜井，340 m） |

辅助功率三条带恒定（16+12+4 kW），提升功率仅与瞬时质量流量有关、与带速无关。**可随调速明显变化的主要为运动功率**。

### 6.2 磨损模型

```
W = W_speed × v² × dt/10 + W_load × Q × v × dt/3600 + W_ramp × |a| × dt
```

- 速度磨损：与速度平方成正比
- 负载磨损：与流量-速度交互成正比
- 加速度磨损：与换挡加速度成正比

### 6.3 累计能耗与节能指标

**单条皮带累计电量**：每仿真步 `ΔE = P(kW) × Δt / 3600`（kWh），从仿真开始累加。

**对照工况**：`Simulator(fixed_speed=True)` 强制各带为 `max_speed`，与智能仿真共享同一入流时间序列。

**综合节能率**：

```
η = max(0, (E_const - E_AI) / E_const × 100)
```

## 7. 预测集成

系统支持两种时序预测后端，切换项为 [config.py](../src/core/config.py) 中的 `PREDICT_BACKEND`（`"timesfm"` / `"chronos"`）。

### 7.1 TimesFM 2.5（默认）

- **Python 包**：TimesFM（需从官方仓库源码安装，见 [requirements-timesfm.txt](../requirements-timesfm.txt)）
- **权重目录**：`models/timesfm-2.5-200m-pytorch/`
- **实现**：[predictor.py](../src/predict/predictor.py) TimesFM 加载与 `forecast_with_covariates` 协变量预测
- **特性**：支持速度协变量（XReg 模式），连续分位数头、翻转不变性、分位数交叉修正

### 7.2 Chronos-2（可选）

Chronos-2 由 `requirements.txt` 直接安装。权重目录 `models/chronos-2/`，若不存在将自动从 HuggingFace 拉取。

**兼容性**：建议 Python 3.10～3.12。若加载失败，`Predictor` 会自动回退到 Chronos-2。

### 7.3 推理流程（两后端共用）

- **触发**：按日志节拍，`Replay` 将最近 60 个采样点送入预测器
- **输出**：未来 10 步流量，含 0.1/0.5/0.9 分位数
- **融合**：仅主运 PID 使用预测前馈（斜井因传输延迟不使用）
- **加权**：预测轨迹按 `1/(1+0.3k)` 衰减，取加权最大值
